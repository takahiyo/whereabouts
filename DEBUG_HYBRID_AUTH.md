# ハイブリッド認証実装 デバッグログ (2026-04-17 更新)

## 現在発生している問題
- **現象**: 
    1. 新規ユーザーがFirebaseで登録・ログインしても、拠点開設画面（`createOffice`）へ遷移しない。
    2. 同期処理は走っているが、以前のセッションの拠点名（`verllion`等）が表示されたまま画面が動かない。
- **調査ログ**:
    - `auth.js` V5: `finalizeLogin: token=false, isBooting=false` が出力されている。
    - `isBooting` が `false` のため、拠点未作成時のリダイレクトロジックがスキップされていた。
    - Worker側の `renew` アクションが `token` を返却していないため、`SESSION_TOKEN` が上書きされて消えていた（`token=false`の原因）。

## 原因の仮説 (2026-04-17 追加)
1. **遷移制限バグ**: `isBooting` フラグが解除された後の Firebase Auth 状態変化（メール認証完了後のリロード等）において、拠点がまだ作られていない場合の遷移処理が実装漏れしていた。
2. **トークン伝搬漏れ**: `renew` アクションが認証トークンをレスポンスに含めていないため、フロントエンドで既存の有効なトークンが `undefined` で上書きされていた。

## 実施した対策とデバッグ強化
1. **[Worker] Auth情報の確実な返却**:
    - `signup` アクションで新規登録成功時もユーザー情報を返すように修正。
    - `renew` アクションでトークンを確実にフロントエンドに返すように修正。
2. **[Frontend] 遷移ロジックの修正**:
    - `auth.js` の `handleFirebaseUser` において、`isBooting` に関わらず拠点がなければ `createOffice` へ遷移するように修正。
    - `finalizeLogin` において、`token` が存在する場合のみ上書きするように堅牢化。
3. **バージョンの引き上げ**:
    - `index.html` および各ファイルのバージョンを `v6` に更新。内容の変更を確実にするためタイムスタンプを更新。
