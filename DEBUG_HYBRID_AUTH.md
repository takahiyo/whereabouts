# ハイブリッド認証実装 デバッグログ (2026-04-17 更新)

## 現在発生している問題
- **現象**: D1ログイン成功後（セッション検証成功後）、ログイン画面は消えるがメインの表（ボード）が表示されず白画面のままになる。
- **調査ログ**:
    - `auth.js` V3: `D1セッションの検証に成功しました` と表示。
    - `sync.js` V2: `Starting sync via Cloudflare Worker.` と表示。
    - `sync.js` V2: `[sync-decision] ... decision: "skip"` と表示。データの差分がないため、描画処理 (`applyState` / `renderBoard`) がスキップされている。

## 原因の仮説 (2026-04-17)
1. **差分なしスキップ問題**: 同期ロジックが「前回のセッション」や「キャッシュ」と比較して「変化なし」と判断すると、描画処理を一切行わない。初回ログイン時やセッション復旧時は、画面が `u-hidden` の初期状態であるため、たとえデータに変化がなくても最低一度は描画を実行して表示を ON にする必要がある。
2. **設定変更なしスキップ問題**: `fetchConfigOnce` においても、拠点設定（`CONFIG_UPDATED`）に変化がないと `render()` をスキップする。これも初回表示を阻害する。

## 実施した対策とデバッグ強化
1. **強制描画フラグ (`FORCE_RENDER_ONCE`) の導入**:
    - `globals.js` に `var FORCE_RENDER_ONCE` を追加。
    - `auth.js` の `finalizeLogin` （ログイン成功時）に `true` をセット。
    - `sync.js` の `pollAction` および `fetchConfigOnce` で、このフラグが `true` の場合は差分に関わらず `render()` / `applyState()` を実行するように変更。
2. **詳細ログの追加**:
    - `board.classList` の遷移状態、同期サイクルの開始、描画関数の呼び出しタイミングに `【DEBUG】` プレフィックス付きのログを追加。
